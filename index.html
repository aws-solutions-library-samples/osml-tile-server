OversightML Tile Server

The OversightML Tile Server is a lightweight, cloud-based tile service which provides RESTful APIs for accessing pixels and metadata for imagery stored in the cloud. Key features include:

  • Works with imagery conforming to the Cloud Optimized GeoTIFF (COG) and National Imagery Transmission Format (NITF) standards

  • Creates both orthophoto map and unwarped image tiles as needed from the source image pyramid

  • Outputs image tiles in PNG, TIFF, JPEG formats and can also output NITFs for tiles without warping

  • Conforms to the OGC API - Tiles specification

REST APIs

Retrieve a list of available viewpoints. Viewpoints are resources that are created to make images stored in the cloud available through these service APIs.

Return:

List of viewpoints with details from the table.

Query Parameters:

  • max_results

  • next_token

Status Codes:

  • 200 OK – Successful Response

  • 404 Not Found – Not found!

  • 422 Unprocessable Entity – Validation Error

POST /viewpoints/

  • viewpoint_request – client’s request which contains name, file source, and range type.

  • request – A handle to the FastAPI request object.

Return:

Status associated with the request to create the viewpoint in the table.

  • 404 Not Found – Not found!

GET /viewpoints/{viewpoint_id}/image/bounds

Get Image Bounds

Get the [min X, min Y, max X, max Y] boundary of the image in pixels. [0, 0] is assumed to be in the upper left corner of the image with x increasing in columns to the right and y increasing in rows down. The boundary coordinates are the upper left and lower right corners of the cropped region.

Parameters:

  • aws – Injected AWS services.

  • viewpoint_id (string, required) – Unique viewpoint id to get from the table.

  • viewpoint_id

Return:

FileResponse containing bounds for the given item.

Status Codes:

  • 200 OK – Successful Response

  • 404 Not Found – Not found!

  • 422 Unprocessable Entity – Validation Error

GET /viewpoints/{viewpoint_id}/image/crop/{min_x},{min_y},{max_x},{max_y}.{img_format}

Get Image Crop

Crop out an arbitrary region of the full resolution image given a bounding box in pixel coordinates. [0, 0] is assumed to be in the upper left corner of the image with x increasing in columns to the right and y increasing in rows down. The [min_x, min_y, max_x, max_y] coordinates are the upper left and lower right corners of the cropped region.

Parameters:

  • aws – Injected AWS services.

  • viewpoint_id (string, required) – Unique viewpoint id to get from the table as a crop.

  • min_x (integer, required) – The left pixel coordinate of the desired crop.

  • min_y (integer, required) – The upper pixel coordinate of the desired crop.

  • max_x (integer, required) – The right pixel coordinate of the desired crop.

  • max_y (integer, required) – The lower pixel coordinate of the pixel crop.

  • img_format (required) – Desired format for cropped output. Valid options are defined by GDALImageFormats.

  • compression – Desired compression algorithm for the output image.

  • width – Optional width in px of the desired crop, if provided, max_x will be ignored.

  • height – Optional height in px of the desired crop, if provided, max_y will be ignored.

  • viewpoint_id – Unique identifier for this viewpoint

  • min_x – The left pixel coordinate of the desired crop.

  • min_y – The upper pixel coordinate of the desired crop.

  • max_x – The right pixel coordinate of the desired crop.

  • max_y – The lower pixel coordinate of the pixel crop.

  • img_format – Desired format for cropped output. Valid options are defined by GDALImageFormats.

Return:

Response of cropped image binary with the appropriate mime type based on the img_format

Query Parameters:

  • compression – Desired compression algorithm for image.

  • width (integer) – Optional. Width in px of the desired crop. If provided, max_x will be ignored.

  • height (integer) – Optional. Height in px of the desired crop. If provided, max_y will be ignored.

Status Codes:

  • 200 OK – Successful Response

  • 404 Not Found – Not found!

  • 422 Unprocessable Entity – Validation Error

GET /viewpoints/{viewpoint_id}/map/tiles/{tile_matrix_set_id}/{tile_matrix}/{tile_row}/{tile_col}.{tile_format}

Get Map Tile

Create a tile by warping the image into an orthophoto and clipping it at the appropriate resolution/bounds for the requested tile. This endpoint conforms to the OGC API - Tiles specification.

Parameters:

  • aws – Injected AWS services.

  • viewpoint_id (string, required) – The viewpoint id.

  • tile_matrix_set_id (string, required) – The name of the tile matrix set (e.g. WebMercatorQuad).

  • tile_matrix (integer, required) – The zoom level or tile matrix it.

  • tile_row (integer, required) – The tile row in the tile matrix.

  • tile_col (integer, required) – The tile column in the tile matrix.

  • tile_format (required) – The desired output format.

  • compression – The desired compression.

  • invert_y – Whether to invert the tile y index.

  • viewpoint_id

  • tile_matrix_set_id

  • tile_matrix

  • tile_row

  • tile_col

  • tile_format – Output image type.

Return:

A Response binary image containing the map tile created from this viewpoint.

Query Parameters:

  • compression – Compression Algorithm for image.

  • invert_y (boolean) – Invert the TMS tile y-index.

Status Codes:

  • 200 OK – Successful Response

  • 404 Not Found – Not found!

  • 422 Unprocessable Entity – Validation Error
